# 리조트 예약 시스템

## 개요

리조트 예약 시스템을 개발한다.

주요 기능은 회원 가입, 로그인, 객실 조회, 객실 예약, 예약 확인이다.
회원은 일반 회원과 분양 회원으로 구분하며, 분양회원번호가 있으면 분양 회원으로 처리한다.
분양회원번호가 없으면 일반 회원으로 처리한다.
회원 유형에 따라 예약 가능한 객실과 객실 요금이 다를 수 있다.

---

## 회원 가입

### 입력 항목

| 항목 | 필수 | 비고 |
|------|------|------|
| 분양회원번호 | X | 입력 시 분양 회원으로 처리 |
| 이름 | O | |
| 연락처 | O | |
| 이메일 | X | |
| 아이디 | O | |
| 비밀번호 | O | |

### 회원 구분 규칙

- 분양회원번호가 존재하면 분양 회원으로 처리한다
- 분양회원번호가 없으면 일반 회원으로 처리한다

### 검증 조건

- 아이디 중복 체크
- 비밀번호 8자리 이상
- 비밀번호는 영문과 숫자를 포함해야 함
- 필수 입력값 누락 검증
- 연락처 형식 검증
- 이메일 입력 시 이메일 형식 검증
- 분양회원번호 입력에 대한 검증은 제외한다. 입력 여부만 판단한다.

## 로그인

- 아이디/비밀번호 로그인
- 로그인 실패 시 오류 메시지 alert 로 표시

---

## 객실

객실 정보는 초기 단계에서 mock 데이터로 제공한다.

### 객실 정보 항목

| 항목 | 설명 |
|------|------|
| 객실코드 | 객실 고유 코드 |
| 객실명 | 객실 표시명 |
| 재고 | 예약 가능 수량 |
| 회원유형별 금액 | 일반/분양 회원에 따라 다른 금액 |
| 예약 가능 회원유형 | 해당 객실을 예약할 수 있는 회원 유형 |

### 비즈니스 규칙

- 객실의 예약 가능 회원유형에 따라 예약 가능/불가가 달라진다
- 객실 상세 페이지에서 일반요금/회원 요금을 표기한다
- 객실 데이터는 관리자 등록 기능 없이 mock 데이터 기반으로 조회만 제공한다

---

## 예약

- 로그인한 회원만 예약 가능
- 예약은 무조건 1박 1실 예약만 가능하다. 
- 예약가능일은 오늘 ~ 6개월 까지의 날짜만 가능하며, date picker 에도 그 날짜만 활성화 되어 있어야 한다.
- 예약 시 예약자 이름과 연락처가 노출되고, 아래 투숙자 이름 연락처 노출칸이 나온다. 예역자와 동일 을 체크하면 자동으로 예약자 정보가 투숙자 정보로 복사해서 들어간다.
- 예약 진행 시 아래 조건을 반드시 검증해야 한다:
  - 해당 회원 유형이 해당 객실을 예약 가능한지
  - 해당 객실의 재고가 예약 가능한 상태인지
- 재고가 0이면 예약 불가 처리해야 한다

---

## 예약 확인

- 로그인한 회원은 본인 예약 내역을 확인할 수 있어야 한다
- 예약 확인 페이지에서는 최소 아래 정보를 제공한다:
  - 예약번호
  - 객실명
  - 투숙자 정보
  - 예약일자
  - 회원유형
  - 예약 상태

---

## 페이지

| 페이지 | 설명 |
|--------|------|
| 메인 | 랜딩 페이지 및 객실 목록 조회 |
| 회원 가입 | 회원가입 폼 |
| 로그인 | 로그인 폼 |
| 객실 상세 | 메인에서 클릭 시 객실 상세 페이지/ 예약 클릭 시 예약폼으로 이동 |
| 예약 | 예약 폼 (로그인 필요) |
| 예약 확인 | 본인 예약 내역 (로그인 필요) |

---

## 권한

- 비회원 접근 불가 페이지가 존재해야 한다
- 예약 및 예약 확인 페이지는 로그인 사용자만 접근 가능해야 한다

---

## 핵심 비즈니스 로직

- 분양회원번호가 있으면 분양 회원, 없으면 일반 회원으로 처리한다
- 객실은 회원 유형별로 예약 가능 여부가 다를 수 있다
- 객실은 회원 유형별 금액이 다를 수 있다
- 예약은 1박 1실만 가능하다
- 예약가능일은 오늘 ~ 6개월 이후까지 인다.
- 예약 시 회원 유형과 객실 재고를 반드시 검증해야 한다
