# 빌드 로그 생성 스킬

## 트리거
프로젝트 생성 자동 흐름의 **마지막 단계(8번)**에서 반드시 실행.

## 목적
빌드 과정 전체를 기록하여 재현성과 개선 포인트를 추적한다.

## 저장 위치
```
example/log/{YYYY-MM-DD}_{명세명}.md
```
예시: `log/2026-04-16_resort-reservation.md`

## 로그 포맷

```markdown
# 빌드 로그 — {명세명}

- **일시**: {YYYY-MM-DD HH:mm}
- **명세 파일**: specs/{명세파일명}
- **총 소요시간**: {시작~완료}
- **총 토큰 사용량**: {입력 토큰 + 출력 토큰}

---

## 단계별 실행 기록

### Step 0: 사전 환경 체크
- **사용 스킬**: skills/precheck.md
- **참조 하네스**: (없음)
- **판단/행동**: {체크 결과, 스킵 여부}
- **소요시간**: {시간}

### Step 1: 백엔드 초기 셋팅
- **사용 스킬**: skills/init-backend.md
- **참조 하네스**: stack.md, structure.md, naming.md, coding.md, architecture.md
- **판단/행동**: {새로 생성 or 스킵, 생성된 파일 수}
- **생성 파일**: {목록}
- **소요시간**: {시간}

### Step 2: 프론트엔드 초기 셋팅
- **사용 스킬**: skills/init-frontend.md
- **참조 하네스**: stack.md, structure.md, style-guide.md, ux.md, coding.md
- **판단/행동**: {새로 생성 or 스킵, 생성된 파일 수}
- **생성 파일**: {목록}
- **소요시간**: {시간}

### Step 3: 기능 구현
- **사용 스킬**: skills/crud-page.md
- **참조 하네스**: 전체 (naming.md, coding.md, architecture.md, ux.md, style-guide.md)
- **참조 교훈**: skills/lessons-learned.md — {적용한 교훈 번호}
- **판단/행동**: {도메인별 생성 내역}
- **생성 파일**: {백엔드 N개, 프론트엔드 N개}
- **소요시간**: {시간}

### Step 4: 검증
- **backend**: `./gradlew build` → {성공/실패}
- **frontend**: `npx tsc -b && npx vite build` → {성공/실패}
- **소요시간**: {시간}

### Step 5: 수정 → 하네스 피드백 (해당 시)
- **발견 문제**: {문제 설명}
- **수정 내용**: {수정 파일, 변경 사항}
- **하네스 반영**: {harness 파일에 추가한 규칙}
- **재검증**: {성공/실패}
- **소요시간**: {시간}

### Step 6: schema.md 업데이트
- **추가 테이블**: {테이블명 목록}
- **소요시간**: {시간}

### Step 7: 완료 보고
- **생성 파일 총 수**: 백엔드 {N}개 + 프론트엔드 {N}개 = 총 {N}개
- **소요시간**: {시간}

---

## 요약

| 항목 | 값 |
|------|-----|
| 총 소요시간 | {MM분 SS초} |

| 검증 재시도 횟수 | {N}회 |

| 사용 스킬 | {목록} |

```

## 작성 규칙

1. **시간 기록**: 각 단계 시작/종료 시점을 기록하여 소요시간 산출
2. **토큰 추정**: 대화 시작~종료까지의 전체 토큰을 추정 기록
   - 하네스/스킬 파일 읽기, 코드 생성, 빌드 출력 등 모든 입출력 포함
3. **판단 근거**: 각 단계에서 왜 그렇게 했는지 (스킵 이유, 적용한 교훈 등)
4. **스킵된 단계**도 기록 (스킵 사유 명시)
5. **마지막에 반드시** 총 토큰과 총 시간을 사용자에게 텍스트로 출력
